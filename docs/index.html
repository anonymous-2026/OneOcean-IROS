Abstract

Simulation is essential for marine robotics, yet existing platforms often trade off between visually realistic worlds, high-throughput experimentation, and ocean-condition realism grounded in real data. OneOcean is a data-grounded simulation suite and benchmark built around a unified spatiotemporal ocean-environment product that harmonizes bathymetry with data-driven currents and tides, with optional pollution fields for cleanup-related evaluation.

On top of this environment representation, OneOcean defines a task ladder spanning navigation and station-keeping under currents, waypoint and route following, depth-profile tracking, area scanning, pipeline inspection, and multi-agent coordination, including formation transit, fish protection/herding, and surface pollution localization, containment, and cleanup. The suite supports teams of 2-10 vehicles, standardizes metrics and run manifests, and exposes meaningful differences among heuristic baselines, behavior-cloning policies, and LLM-based planners under realistic, data-grounded disturbances.

Overview

Overview figure from the paper. The repository also keeps the original vector version at static/images/paper/overview_abstract.pdf.

OneOcean treats ocean conditions as a reusable product. A unified NetCDF environment dataset harmonizes GEBCO bathymetry, Copernicus/GOPAF currents, optional tide terms, and derived masks. The same grounded environment then drives a canonical benchmark suite instead of being tied to a single simulator implementation.

The benchmark is executed in two complementary modes. The core benchmark backend prioritizes deterministic, throughput-first, paper-ready evaluation. The external underwater-scene backend reuses packaged underwater scenes while injecting the same data-grounded currents, so qualitative scene evidence and quantitative results still share task semantics and provenance.

Data: unified ocean-environment product with currents, bathymetry, masks, and task-grounding metadata.
Tasks: 10 canonical tasks spanning navigation, sensing, coordination, and pollution-response scenarios.
Evaluation: shared metrics, tiered difficulty, auditable manifests, and support for 2-10 vehicles.

Method and Benchmark Design

Unified Environment Product

The data pipeline produces a combined environment dataset with latitude, longitude, time, depth, currents, elevation, and validity information. Variants support fast local iteration, scene construction, and public release without changing task semantics.

Canonical Task Ladder

OneOcean defines a 10-task ladder covering goal reaching, station keeping, route following, depth tracking, area scan, leak inspection, formation transit, fish protection/herding, and pollution localization/containment/cleanup.

Cross-Backend Execution

A throughput-first core backend generates large, reproducible experiment tables, while a scene-backed underwater backend reuses external scenes and injects the same data-grounded currents for qualitative validation in realistic underwater-looking environments.

Navigation

Go-to-goal, station keeping, route following, depth-profile tracking.

Sensing

Area scan and pipeline inspection under drift and terrain constraints.

Coordination

Formation transit and fish protection/herding with multi-agent constraints.

Pollution Response

Localization, containment, cleanup, and underwater lift tasks grounded by task probes.

Highlighted Results

The core benchmark backend is designed to be non-saturated: Tier 2 remains useful for routine evaluation, while Tier 3 exposes large gaps in disturbance rejection, coordination, and task completion. This is exactly the behavior needed from a benchmark that is supposed to separate planning and control quality instead of collapsing to all-success or all-failure regimes.

The results below show three distinct capabilities: (i) the benchmark separates strong heuristics from learned policies on the full task ladder, (ii) planning-sensitive tasks expose meaningful differences among local LLM planners and behavior cloning, and (iii) the same task semantics remain useful when moved into packaged underwater scenes with data-grounded current injection.
