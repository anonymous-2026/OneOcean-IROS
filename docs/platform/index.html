Practical guide for using the OneOcean data pipeline and environment/task stack.

Quickstart

Run data generation and the first navigation task with local variants.

python Data_pipeline/run_pipeline.py --overwrite
python Data_pipeline/generate_variants.py --which tiny,scene,public --overwrite
python -m benchmark_core.cli.run --task go_to_goal_current --variant tiny --episodes 3 --out runs/benchmark_core/quickstart

Dataset Tiers

  • tiny: fast iteration and sanity checks
  • scene: main simulation/task construction tier
  • public: distribution-oriented, larger coverage

Task Development Flow

  1. Load variant and resolve schema contract.
  2. Select a backend: core benchmark or the scene-backed OceanGym path.
  3. Run task with fixed seed and export metrics.

Integration Hooks

Current integration points exposed by the released repository.

  • Project page: paper-facing demos and highlighted benchmark media
  • Benchmark core: quantitative experiments and paper tables
  • Data pipeline: combined dataset generation and variant export
